# AI_WORKBENCH

## Português

AI_WORKBENCH é uma estrutura de governança e engenharia de software orientada por Inteligência Artificial, criada para organizar, controlar e padronizar o desenvolvimento de software assistido por agentes de IA.

O objetivo do projeto é transformar agentes de IA em executores disciplinados de arquitetura, requisitos e processos previamente definidos, reduzindo improvisações, retrabalho e divergências estruturais durante o desenvolvimento.

A Workbench atua como uma “fonte de verdade” externa ao projeto, contendo:
- documentação
- skills
- checklists
- templates
- regras de engenharia
- padrões arquiteturais
- fluxos operacionais
- critérios de validação

A proposta central é simples:

> IA não deve inventar arquitetura durante a codificação.
> A IA deve executar uma engenharia previamente definida.

## Objetivos

- Padronizar desenvolvimento orientado por IA
- Reduzir decisões improvisadas
- Melhorar governança de projetos
- Criar fluxos reproduzíveis
- Separar arquitetura da implementação
- Criar auditoria e rastreabilidade
- Permitir múltiplos agentes especializados
- Transformar prompts em contratos operacionais

## Como Funciona

O fluxo da AI_WORKBENCH é baseado em etapas:

1. Entrevista com humano
2. Levantamento de requisitos
3. Arquitetura
4. Engenharia de processo
5. Geração de documentação
6. Criação de skills e checklists
7. Implementação controlada por IA
8. Validação
9. Consolidação

Os agentes de IA devem sempre ler:
- documentação da etapa
- skills
- regras da workbench
- checklists
- critérios de aceite

antes de qualquer implementação.
